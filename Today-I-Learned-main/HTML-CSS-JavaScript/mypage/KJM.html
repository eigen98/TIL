🍏23주간의 학습 계획

다른 일정보다 앱스쿨 과정을 우선순위로 함.

진도가 다른 경우 우선순위는 공통진도를 우선으로 예습 복습.

진도보다 뒤쳐지는 경우 그만큼 밤새 시간을 투자.

여유가 생기는 경우 동료들이 가진 관점과 생각들을 이해해보는 시간으로 할애

프로그래밍 기초 파트에서 하고 싶은 것

  • HTML,CSS:
    홈페이지를 만들어보면서 HTML과 CSS 숙련도가 부족하단 것을 깨달았기에 컴포넌트 하나하나에 대한 이해도를 높이는 것을 목표로함. XML을 이용해 쉽게 레이아웃을 작성하는 것처럼 같은 수준이 되도록 연습.

  • 자료구조, 알고리즘:
    미흡한 개념(적용을 못하거나 혼자 설명을 못하는 부분)을 위주로 집중 학습. 매주 10문제 이상 알고리즘 문제 풀이를 하며 개발블로그 업로드.

  • Git / Github:
    GUI에 의존해서 깃을 사용하게 되는 경향을 줄이기. 깃허브를 단순히 사용하는 것을 넘어 개념 바로잡기.

  • 네트워크 기본 개념:
    개인적으로 시간이 남는다면 대학강의를 통해 복습해 보고자 한다. 시간이 남는 경우에 실천.

앱 개발 기초에서 하고 싶은 것

  • CI/CD :
    흐릿하게 알고있는 CI/CD라는 개념을 내 언어로 설명할 수 있는 수준으로 이해해보기. CI를 협업때 막연하게 사용했던 것을 다시 뜯어보면서 이해하기.
    •

  • Swift 기초 / 기초 앱 개발 :
    escaping closure,
    weak self,
    Optional,
    Struct와 Class의 차이점, 함수형 프로그래밍 언어와 객체형 프로그래밍 언어의 차이점 이해 및 암기
    등 그저 사용하기 바쁜 문법들에 대해 이해하고 설명해보기

  • Human Interface Guideline :
    HIG가 무엇인지는 알지만 어떤식으로 접근해야할지 다루어야할지 모호함을 해결하고 싶음. 지난번 앱스토어 심사에서 언급된 HIG 문제를 다시 보면서 이해하기.

  • UIKit :
    현재 UIKit 사용에 있어서 잘못된 것이 있는지 미흡한 부분이 있는지 수업과 다른 사람들의 방식을 보고 파악.

앱 개발 심화에서 하고 싶은 것

  • SwiftUI : 현재 경험한적 없는 SwiftUI 사용해보기. SwiftUI가 UIKit으로부터 자유로울 수 없는 이유 파악.

  • XCTest / TDD : 단위테스트, TDD 등 뜻만 알고 있는 상황에서 구체적인 과정의 이해와 직접 적용해보는 경험해보기.

  • 앱 개발 : 현재 머릿속에만 있는 앱의 청사진을 적용하므로서 현재까지의 학습 내용 복습과 개인 토이프로젝트 경험 쌓기.

  • 앱 등록정보 준비하기 : 최근 어렵다고 느낀 앱심사 과정(Bidit앱)에서 생긴 공포심으로부터 벗어나 애플과 친해지기.

최종 프로젝트에서 하고 싶은 것

  • 전체 해커톤 :
    지금은 SwiftUI로 해커톤에 사용해보는 것을 목표로함. 짧은 시간안에 진행되는 해커톤은 경험이 거의 없기에 반드시 참가해보기.

  • 이력서 워크샵 :
    서류 통과와 면접등의 기회는 몇 번 있었지만 구체적으로 어떤 부분에서 어필이 잘 되고 있는지 어떤 부분이 미흡한지 객관화가 필요.

  • 서비스형 프로젝트 고도화 및 배포 :
    기능을 구현하기 급급한 프로젝트가 아닌 사용되고 있는 기술의 본질을 이해하고 적용해보는 프로젝트를 만들어보기.
