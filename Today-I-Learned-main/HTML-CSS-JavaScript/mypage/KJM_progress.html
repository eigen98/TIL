만든 과정

이 웹페이지를 꾸밀 때 어떤 목표를 갖고 시작했는지

  • HTML,CSS, javascript의 기본적인 이해:
    모바일 개발에 집중하다보니 기억이 거의 나지 않는 상황이라 복습하는 시간을 가짐.

  • 간단한 자기소개 및 사진 넣기

  • 23주간의 학습계획을 이쁘게 구획을 나누어 작성(좌우 화면 분할)
  • 체크박스를 사용해 계획표 작성
  • 말풍선 등의 다양한 레이아웃을 CSS로 꾸미기

  • 그동안 경험한 프로젝트 정보 넣기
  • 버튼 클릭 및 화면 전환 기능 넣기

목표를 이루기 위한 진행과정에서 어려운점과 극복과정

  • HTML,CSS 숙련도 부족:
    10시간정도 신속하게 강의를 보면서 자주쓰이는 기본개념들을 기술블로그에 정리하는 시간을 가지고 홈페이지 제작을 하면서 연습함으로서 숙련도를 올림.

  • 태그들의 짝을 찾기가 너무 어려움.
    구글링을 통해 태그들의 짝을 찾아주는 vsCode의 플러그인을 찾아서 해결.

  • 시간의 부족함.
    일단 우선적으로 넣어줄 정보들을 다시 생각하고 우선순위를 정하여 가능한 목표들을 다시 정하였습니다. 수면시간을 줄여 개발 시간을 더 확보하고 부트스트랩을 통해 시간을 절약하였습니다.

  • 이해하지 못하고 사용하게 되는 기능들이 자주 발생.
    부트스트랩이나 가끔 쓰이는 태그들을 완전히 이해하지 못한 상태에서 개발진행하는 경우 잠시 쉬는시간을 가지면서 블로그에 정리.

완성된 웹페이지를 볼 때 목표의 몇 퍼센트가 구현되었다고 생각하는지

  • 완성도 70%:
    기본적인 필수로 들어가야하는 정보는 보여주고 있기에 70%정도 완성되었다고 생각합니다. 하지만 체크박스나 말풍선등 풍부하게 CSS를 적용하지 못하였고 급하게 구현하다보니 정돈되지 않은 태그들로 검색시 최적화등의 문제가 남아있습니다. 또한 경험한 내용도 자세하게 표시하지 못하였습니다.

시간이 더 주어진다면 어떤 점을 보강하고 싶은지

  • 태그들의 무분별한 사용 정리:
    가독성이 떨어지게 만드는 태그들을 다시 정리하고 싶습니다.

  • 계획표 디자인 수정:
    단순하게 나열한 형태인 계획표를 다른 디자인을 참고하여 조금 더 이쁘게 꾸미고 싶습니다.

  • 제 스스로를 어필할 수 있는 정보 추가:
    단순하게 프로젝트를 이미지로 보여주고 있는 형태에서 숙련된 기술스택과 경험해본 기술스택으로 나누고 대외활동과 경력을 나타내는 구획으로 나누는 등의 디테일한 부분에 신경을 쓰고싶습니다.

  • 자바스크립트를 통한 기능 구현:
    끝내 자바스크립트를 학습하고 기능 구현에 사용하지 못하였기에 시간이 주어진다면 메일 보내기 , 버튼 클릭 효과 등의 기능을 구현해보고싶습니다.

사용하는 OS 환경

  • 운영체제:
    mac OS Monterey 12.5.1

  • MacBook Pro(13-inch, M1, 2020)

  • Apple M1 칩
    GUI에 의존해서 깃을 사용하게 되는 경향을 줄이기. 깃허브를 단순히 사용하는 것을 넘어 개념 바로잡기.

  • 메모리 16GB
